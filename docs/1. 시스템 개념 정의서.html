문서 개요목적본 시스템의 목적은 카셰어링 차량의 고정 대여·반납 위치 문제를 관제 센터 기반 원격 주행 제어로 해결할 수 있음을 검증하는 것이다.

기존 카셰어링은 사용자가 차량이 있는 장소까지 이동하고, 이용 후 다시 지정된 장소에 반납해야 하는 불편이 있다. 본 프로젝트는 이를 해결하기 위해 관제 센터의 원격 운전자가 차량을 직접 조종하여 사용자 위치까지 차량을 호출하고, 이용 후 차량을 다시 회수·주차하는 서비스 모델을 목표로 한다.
6
범위본 시스템은 카셰어링 차량 원격 호출·회수 서비스의 핵심 제어 구조를 RC카 기반으로 검증하는 교육용 PoC 시스템이다. 실제 도로 주행이나 상용 서비스를 구현하는 것이 아니라, 관제센터 역할의 노트북에서 입력한 원격 조작 명령이 ECU와 CAN 네트워크를 거쳐 차량의 구동·조향·제동·등화 장치로 전달되는 흐름을 구현하는 데 초점을 둔다.

노트북 키보드는 원격 운전 조작 장치로 사용되며, 전진, 후진, 정지, 좌회전, 우회전, 기어 변경, 전조등, 방향지시등, 비상등 명령을 생성한다. 이 명령은 PC-ECU 통신을 통해 차량 제어 ECU로 전달되고, 각 ECU는 CAN 메시지를 기반으로 모터, 서보, LED 등 액추에이터를 제어한다.

또한 본 시스템은 단순 원격 조종에 그치지 않고, 허용된 CAN ID만 처리하는 CAN ID Filter, 상태 피드백을 위한 ADC 값 취득, 조향·제동을 위한 Servo 위치 제어, 긴급 제동 명령 우선 처리를 검증하는 Starvation Test, 기능별 통신 분리를 위한 Multi-Bus 구조, CAN 프레임 기반 명령 라우팅, CAN과 CAN FD의 통신 성능 비교를 포함한다.
7
시스템 개념개발 배경카셰어링 서비스는 차량을 소유하지 않고도 필요한 시간에 차량을 이용할 수 있다는 장점이 있지만, 현재 대부분의 서비스는 차량이 배치된 지정 주차 구역까지 사용자가 직접 이동하고, 이용 후에도 정해진 장소에 다시 반납해야 하는 구조를 가진다. 이로 인해 사용자의 현재 위치와 차량 위치가 멀거나, 목적지와 반납 장소가 다른 경우 서비스의 편의성이 크게 떨어진다.

특히 비가 오거나 짐이 많은 상황, 심야 시간, 대중교통 접근성이 낮은 지역에서는 차량을 가지러 가는 과정 자체가 부담이 된다. 또한 운영자 입장에서도 차량이 특정 지역에 몰리거나 수요가 낮은 곳에 방치되면 차량을 다시 재배치해야 하므로 인력, 시간, 운영 비용이 증가한다. 즉, 카셰어링 서비스는 사용자의 접근성 문제와 운영자의 차량 재배치 문제를 동시에 가지고 있다.

최근 해외에서는 이러한 한계를 해결하기 위해 관제센터의 원격 운전자가 차량을 사용자 위치까지 이동시키고, 이용 종료 후 다시 원격으로 회수하는 방식의 원격 운전 기반 모빌리티 서비스가 등장하고 있다. 이는 완전 자율주행이 모든 환경에서 상용화되기 전까지, 사람이 원격으로 차량을 제어해 차량 호출과 회수 문제를 해결하는 현실적인 대안이 될 수 있다.

본 프로젝트는 이러한 흐름을 바탕으로, 카셰어링 차량이 사용자에게 찾아가고 이용 후 다시 회수되는 서비스의 핵심 제어 구조를 RC카 환경에서 검증하고자 한다. 노트북을 관제센터 조작 장치로 사용하고, 입력된 원격 명령이 ECU와 CAN 통신을 거쳐 구동, 조향, 제동, 등화 장치로 전달되는 과정을 구현한다.

따라서 본 프로젝트는 단순한 RC카 원격 조종이 아니라, 미래 카셰어링 서비스에 적용될 수 있는 관제센터 기반 원격 차량 호출·회수 시스템의 축소형 제어 모델을 구현하는 데 개발 배경이 있다.
8
9
개발 필요성본 시스템은 카셰어링 서비스의 접근성 문제와 차량 재배치 문제를 원격 주행 제어 기술로 해결할 수 있는지 검증하기 위해 필요하다.

기존 카셰어링 서비스는 사용자가 차량이 있는 위치까지 직접 이동하고, 이용 후에도 지정된 장소에 다시 반납해야 한다. 이로 인해 차량 위치가 멀거나 반납 장소가 목적지와 다를 경우 서비스 편의성이 크게 떨어진다. 원격 호출·회수 시스템이 적용되면 관제센터에서 차량을 사용자 위치까지 이동시키고, 이용 종료 후 다시 회수할 수 있어 사용자의 이동 부담을 줄일 수 있다.

운영자 입장에서도 차량 재배치 문제를 해결할 수 있다. 카셰어링 차량은 수요 지역과 시간대에 따라 위치 불균형이 발생하기 쉬우며, 기존에는 이를 사람이 직접 이동시켜야 했다. 원격 회수 기능을 활용하면 차량을 중앙 관제 방식으로 필요한 위치에 재배치할 수 있어 차량 가동률을 높이고 운영 비용을 줄일 수 있다.

또한 원격 차량 제어를 위해서는 단순 조종이 아니라, 관제 명령이 차량 내부 ECU 네트워크를 통해 안정적으로 전달되고 각 제어 장치가 정확히 동작하는 구조가 필요하다. 본 프로젝트는 PC-ECU 통신, ECU-ECU 간 CAN 통신, 구동·조향·제동·등화 제어, 상태 피드백을 구현함으로써 원격 차량 제어에 필요한 차량 내부 통신 구조를 검증한다.

원격 제어 시스템에서는 비정상 명령, 통신 지연, 통신 단절, 우선순위 충돌이 발생할 수 있기 때문에 안전 제어 구조도 반드시 필요하다. 본 시스템은 CAN ID 필터링, 긴급 정지, 비정상 메시지 무시, 통신 이상 시 정지 상태로 전환하는 Fail-Safe 구조를 포함하여 원격 차량 제어에서 안전성이 왜 중요한지 보여준다.

따라서 본 프로젝트는 단순 RC카 조종이 아니라, 카셰어링 서비스 문제 정의, 원격 주행 기반 해결책, CAN 기반 차량 제어 구조, 안전 검증 시나리오를 하나로 연결한 교육용 PoC 시스템이라는 점에서 개발 필요성이 있다.
10
문제 정의현재 카셰어링 서비스의 핵심 문제는 사용자가 원하는 위치에서 차량을 바로 이용하고, 원하는 위치에서 편리하게 반납하기 어렵다는 점이다. 차량은 대부분 지정된 주차 구역에 배치되어 있기 때문에 사용자는 차량이 있는 곳까지 직접 이동해야 하며, 이용 후에도 정해진 장소에 다시 반납해야 한다.

이로 인해 사용자에게는 접근성 저하가 발생한다. 카셰어링이 “필요할 때 바로 이용하는 서비스”가 아니라, “차량이 있는 곳까지 찾아가야 하는 서비스”가 되어 편의성이 떨어진다. 운영자 입장에서도 차량이 특정 지역에 몰리거나 수요가 낮은 곳에 방치되면 다시 재배치해야 하므로 인력, 시간, 비용이 발생하고 차량 가동률도 낮아진다.

이를 해결하기 위해서는 관제센터에서 차량을 원격으로 호출·회수할 수 있는 구조가 필요하다. 그러나 원격 차량 제어를 위해서는 단순 조종뿐 아니라, 원격 명령이 차량 내부 ECU 네트워크로 안정적으로 전달되고, 구동·조향·제동·등화 장치가 충돌 없이 동작하며, 비정상 명령이나 통신 이상 발생 시 안전하게 정지할 수 있어야 한다.

따라서 본 프로젝트의 문제는 카셰어링의 고정 대여·반납 위치 문제를 해결하기 위해, 관제센터의 원격 명령을 CAN 기반 ECU 네트워크로 전달하고 RC카의 주행, 조향, 제동, 등화 장치를 안전하게 제어할 수 있는 원격 호출·회수형 차량 제어 시스템이 필요하다는 것으로 정의한다.
11
시스템 목표본 시스템의 목표는 관제센터에서 입력한 원격 주행 명령이 차량 내부 ECU와 CAN 통신을 통해 RC카의 구동·조향·제동·등화 장치로 안정적으로 전달되고 제어되는 구조를 구현하는 것이다.

이를 위해 노트북 키보드를 관제센터 조작 장치로 사용하여 전진, 후진, 정지, 좌회전, 우회전, 기어 변경, 전조등, 방향지시등, 비상등, 긴급 정지 명령을 입력할 수 있도록 한다. 입력된 명령은 PC-ECU 통신을 통해 차량 제어 ECU로 전달되고, ECU는 이를 CAN 메시지로 변환하여 각 제어 장치에 분배한다.

차량 내부에는 구동, 조향, 제동, 등화, 상태 관리 기능을 담당하는 ECU가 존재한다고 가정하고, ECU-ECU 간 CAN 통신을 통해 명령과 상태 정보를 주고받는 분산 제어 구조를 구현한다. 또한 CAN ID 필터링을 적용하여 허용된 메시지만 처리하고, 비정상 ID나 알 수 없는 명령은 무시하거나 오류 상태로 전환한다.

구동 모터, 서보 모터, LED를 활용하여 원격 명령이 실제 RC카의 전진·후진, 조향·제동, 전조등·방향지시등·비상등 동작으로 이어지도록 구현한다. 동시에 차량의 주행 상태, 조향 상태, 제동 상태, 등화 상태, 센서 또는 ADC 값을 다시 관제 장치로 전달하여 상태 피드백 구조를 만든다.

또한 Frame Based Routing을 통해 수신한 CAN 프레임을 기능별 제어 로직으로 분배하고, 긴급 정지 명령은 일반 명령보다 우선 처리되도록 설계한다. Multi-Bus 또는 Dual Line Topology 구조를 적용하여 제어 명령과 상태 피드백, 일반 기능과 안전 기능을 분리하는 구조도 검토한다.

통신 안정성 검증을 위해 Starvation Test를 수행하여 여러 메시지가 동시에 발생하는 상황에서도 제동, 조향, 긴급 정지와 같은 중요 메시지가 지연 없이 처리되는지 확인한다. 또한 CAN과 CAN FD의 통신 속도 및 데이터 전송 효율을 비교하여 원격 차량 제어 시스템에서 CAN FD의 확장 가능성을 분석한다.

최종적으로 통신 단절, 비정상 명령, 긴급 정지 상황에서 차량이 안전하게 정지하는 Fail-Safe 동작을 구현하고, 차량 호출 → 원격 이동 → 사용자 이용 → 반납 요청 → 원격 회수·주차로 이어지는 카셰어링 원격 호출·회수 시나리오를 시연하는 것을 목표로 한다.
12
기대 효과본 시스템의 기대효과는 카셰어링 서비스의 사용자 편의성 향상, 운영 효율 개선, 원격 차량 제어 기술 검증, 안전 제어 구조 확보이다.

첫째, 사용자는 차량이 있는 곳까지 직접 이동하지 않고도 원하는 위치에서 차량을 받을 수 있으며, 이용 후에도 지정 반납 장소까지 이동하지 않아도 된다. 이를 통해 카셰어링 서비스의 접근성과 편의성을 높일 수 있다.

둘째, 운영자는 차량을 원격으로 회수하거나 재배치할 수 있어 차량 위치 불균형 문제를 줄일 수 있다. 이는 차량 재배치에 필요한 인력과 시간을 줄이고, 차량 가동률과 운영 효율을 높이는 효과로 이어질 수 있다.

셋째, 본 프로젝트는 완전 자율주행이 상용화되기 전 단계에서 활용 가능한 관제센터 기반 원격 운전 서비스 모델을 RC카 환경에서 검증한다. 이를 통해 미래 카셰어링 서비스에서 차량 호출·회수 기능이 어떻게 구현될 수 있는지 보여줄 수 있다.

넷째, PC-ECU 통신, ECU-ECU 간 CAN 통신, CAN ID Filter, Frame Based Routing, Multi-Bus, Starvation Test 등 강의 핵심 기술을 하나의 서비스 시나리오 안에 통합하여 실전형 차량 통신 시스템으로 구현할 수 있다.

다섯째, 비정상 명령 무시, 긴급 정지, 통신 이상 시 정지 상태 전환과 같은 Fail-Safe 구조를 적용함으로써 원격 차량 제어에서 안전성이 왜 중요한지 명확하게 보여줄 수 있다.

따라서 본 프로젝트는 단순한 RC카 조종이 아니라, 카셰어링 문제 해결 시나리오, 원격 운전 서비스 모델, CAN 기반 ECU 제어, 안전 검증, 시연 가능성을 모두 갖춘 프로젝트로서 기술성과 완성도를 동시에 보여줄 수 있다.
